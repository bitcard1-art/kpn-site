자문위원을 등록·수정하려면 관리자 비밀번호가 필요합니다.

자문위원 등록 가이드

KPN의 "검증된 판단"은 위원의 신원이 진본임을 보증하는 데서 출발합니다. 새 위원은 아래 절차를 거쳐야 자문 라우팅에 포함됩니다.

① 자격·동의 결정→ ② MIP에서 DID 발급→ ③ 화이트리스트 등록→ ④ 위원 등록 폼 저장→ ⑤ 라우팅 확인
PRE

0 · 위원 자격과 본인 동의 확정

담당: 운영자(의장) 판단

등록 전에 먼저 정합니다. 이 단계는 기술이 아니라 운영 판단입니다.

  • 보증 분야 — 이 위원이 어떤 영역의 판단을 보증하는가 (예: 한국 IT 영업·파트너십)
  • 책임 구조 — 위원의 판단이 들어간 리포트가 틀렸을 때의 책임. DID 서명은 곧 책임 표시입니다.
  • 본인 동의 — 실명·DID·판단이 고객 리포트에 박히고 공개 사이트에 노출됩니다. 본인이 명시적으로 동의했는지 확인하세요.
자격·동의가 확정되지 않은 페르소나는 등록하지 마세요. 검증된 판단의 신뢰는 여기서 시작됩니다.
1

1 · MIP에서 페르소나 DID 발급

담당: 위원 본인 / MIP(mio.mysoma.space)

위원이 되려면 그 사람의 페르소나가 MIP에서 DID를 발급받고, Lore·DNA가 채워진 상태여야 합니다. MCP 진본 검증(VALID)은 이 DID를 기준으로 이뤄집니다.

  • MIP에서 페르소나 생성 → DID 발급 (형식: did:mip:persona:... 또는 did:persona:...)
  • Lore 연결 · DNA 완료 · MCP 동기화 "활성" 상태 확인 (위원 관리 탭의 MCP 연동 현황에서 확인 가능)
이 DID가 없거나 검증이 VALID가 아니면, 이후 단계의 등록 폼이 저장을 거부합니다.
2

2 · 등록 화이트리스트에 DID 추가

담당: 개발(백엔드)

현재 등록 폼은 보안상 승인된 DID만 저장을 허용합니다(기본: 의장 DID만). 새 위원의 DID를 백엔드 화이트리스트에 추가해야 진본 검증을 통과합니다.

  • persona-mcp-server의 위원 등록 허용 목록(화이트리스트)에 새 DID 추가
  • 그 DID가 MCP 진본 검증 VALID를 통과하는지 확인
이 게이트가 더미·테스트 페르소나가 위원으로 등록되는 것을 막습니다. 우회하지 마세요.
3

3 · 위원 등록 폼 작성·저장

담당: 운영자 · "위원 관리" 탭

화이트리스트가 열리면 위원 관리 탭의 등록 폼을 채웁니다.

  • 이름 / 직함 / 분야 — 0단계에서 정한 보증 분야
  • 신뢰도 점수 / 자문 이력 — 리포트 신뢰도 표기에 반영
  • 경력·전문성 — 고객 리포트와 공개 사이트에 노출되는 한 줄
  • 페르소나 DID — 1단계에서 발급한 그 DID
  • 공개 사이트 표시 — 체크 시 홈페이지 위원 소개에 실명 노출

저장(검증 후 등록)을 누르면 MCP 진본 검증을 시도하고, VALID일 때만 등록됩니다.

4

4 · 자문 라우팅 포함 확인

담당: 개발 · 운영자

등록됐다고 자동으로 자문에 불리는 것은 아닙니다. 새 위원이 실제 교차검증 라우팅 후보에 포함되는지 테스트 자문 1건으로 확인하세요.

  • 테스트 접수 → 새 위원이 council에 라우팅되는지 확인
  • 교차검증은 위원 2명 이상일 때 의미가 있습니다(1명이면 "교차검증 제한적"으로 표기됨)
여기까지 통과하면 새 위원의 판단이 실제 자문 리포트에 반영됩니다.

위원 등록

DID를 입력하면 저장 시 페르소나 MCP 서버로 진본 검증(VALID)을 시도합니다.

현재는 이영도 위원장(did:mip:persona:lee-youngdo)만 등록 가능합니다. MCP 진본 검증(VALID)을 통과해야 저장됩니다.
저장소에 올린 파일명(youngdo.jpg)이나 외부 이미지 URL을 넣으세요.
